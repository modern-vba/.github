# modern-vba

**modern-vba** is a home for tools and reusable modules that make VBA development more maintainable, testable, and automation-friendly.

VBA is still used in many real business systems, especially around Excel. This organization focuses on improving that development experience with source-controlled modules, repeatable build workflows, documentation tools, and shared libraries.

## Projects

- **vba-tools**  
  Command-line tooling for working with VBA projects, including workflows for importing, exporting, building, and testing Excel macro workbooks.

- **xls-common-modules**  
  Reusable VBA modules for Excel macro projects. These modules provide shared building blocks that can be distributed across multiple workbooks and repositories.

- **DoxyVB6**  
  A documentation support tool that works as an input filter for Doxygen, helping generate documentation from VB6/VBA-style source code.

## Goals

- Keep VBA source code under version control.
- Make Excel macro projects easier to build, test, and review.
- Support reusable modules across multiple workbook-based projects.
- Improve documentation and maintainability for long-lived VBA systems.
- Bridge practical business automation with modern development workflows.

## Focus

This organization is especially interested in:

- Excel VBA project structure
- VBA module import/export workflows
- Automated testing for macro-enabled workbooks
- Documentation generation for VBA and VB6-style code
- Tooling that helps legacy VBA codebases evolve safely
